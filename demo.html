Technical Implementation

FSRS-6 Algorithm

  • Stability (S): Time in days until card is likely to be forgotten
  • Difficulty (D): 1-10 scale indicating card difficulty
  • Interval: Days until next review, calculated from stability
  • Lapses: Count of times the card was forgotten
  • Binary Grading: Simplified to Again (failed) or Good (passed)

Data Persistence

  • User Data: Stored in data/users/{username}/{deck_name}/
  • Card Metadata: JSON file with FSRS-6 parameters per card
  • Deck Metadata: JSON file with daily limits and progress
  • CSV Updates: hiragana.csv updated with state and lastSeen

File Structure

  • flashcard_app.py - Main application controller
  • models.py - Data models (Card, DeckMetadata)
  • fsrs.py - FSRS-6 scheduler implementation
  • persistence.py - Data persistence layer
  • gui.py - Tkinter GUI components
  • test_app.py - Unit tests
